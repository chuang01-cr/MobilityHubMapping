# Data Object Definitions

### Facility Data

| What? | Source | Fields | Color | Notes |
| :---- | :---- | :---- | :---- | :---- |
| Transit Stops | GTFS feeds, with urls compiled by [TransitLand](https://www.transit.land/) and downloaded directly from agencies | **Stop ID** \- the GTFS stop id, **Minimum Headway** \- the minimum of the headway for each set of overlapping service patterns through the stop, **Total Frequency** \- the total frequency in both directions at the stop **Mode** \- the mode of the routes serving the stop. If routes with different modes serve the stop, the order is determined by the ranking in `ROUTE_PRIORITY_MAP` **Mode Classification \-** “High Comfort” if the stop is served by non-cable car  rail or ferry, “Bus” if the stop is only served by buses, trolleybuses, or cable cars, “Other” if an unusual mode serves the stop and it is not served by a high comfort mode. **Transfer** \- whether the stop is identified as a transfer. A transfer is defined as a stop with more than two service patterns if clustering is enabled, or one service pattern if clustering is disabled | Pink if local, Blue if destination, Dark if trunk, Light if branch, Grey if not a mobility hub | There seem to be some issues still to be worked out with clustering and transfer detection |
| Mobility Hubs | Transit Stop and Block Group data | **Origin / Local?** \- Whether the stpo is classified as a local stop or an origin stop **Origin / Local Score** \- The score used to determine whether a stop is an origin or local stop **Trunk / Branch?** \- Whether the stop is classified as a trunk, branch, or non-mobility-hub stop **Mode** \- “High Comfort” if the stop is served by non-cable car  rail or ferry, “Bus” if the stop is only served by buses, trolleybuses, or cable cars, “Other” if an unusual mode serves the stop and it is not served by a high comfort mode. **Minimum Headway** \- the minimum of the headway for each set of overlapping service patterns through the stop  **Total Frequency** \- the total frequency in both directions at the stop **Transfer** \- whether the stop is identified as a transfer. A transfer is defined as a stop with more than two service patterns if clustering is enabled, or one service pattern if clustering is disabled | Varies based on whether the stop is origin or local |  |
| Bikeshare Stations | The [CityBikes](https://api.citybik.es/v2/) API which is based on operator provided GBFS feeds | **System Name** \- the name of the system **Operator** \- the organization (typically a contractor) that operates the bikeshare system **Station Name** \- the name of the station **Capacity** \- the number of bikes that can be parked at the station **Has Ebikes?** \- true if ebikes are provided by the system and able to be docked at the station | Green |  |
| Bike Parking | [OpenStreetMap](https://www.openstreetmap.org/) | **Facility Type** \- the type of bike parking, typically either “stands” or “lockers” **Capacity** \- the number of bikes that can be parked **Covered?** \- whether the parking is covered  | Red | OSM’s bike parking map is generally not comprehensive, and often has missing metadata |
| Public EV Chargers | NREL [Alternative Fuels Data Center](https://developer.nrel.gov/docs/transportation/alt-fuel-stations-v1/) | **Network** \- the operator of the station **Website** \- the operator’s website **Name** \- the station name **Address** \- the station’s street address | Blue | May have data quality issues (shows a charger in the middle of Silver Lake Reservoir) |
| Transit Agency Owned Parking | FTA NTD 2022 [Annual Database Facility Inventory](https://www.transit.dot.gov/ntd/data-product/2022-annual-database-facility-inventory) | **Facility Name** \- the name of the facility in the FTA database **Facility Type** \- whether it is a surface or multi-story lot **NTD ID** \- the NTD ID of the owning agency **Notes** \- any notes in the NTD souce | Light Blue | Database is somewhat outdated and at least in LA has major quality issues (some locations have very basic typos in their address and no lat/lon) |
| Bike Infrastructure | [OpenStreetMap](https://www.openstreetmap.org/) | **Cycleway Type** \- the value for “cycleway” from OSM, either “lane” or “track” (protected lane) **Road Type** \- the value for “highway” from OSM, could be “secondary road”, “path”, etc. **Paint Only?** \- Whether the infrastructure is a paint only bike lane based on OSM | Light green (paint only), or dark green (protected or separate path) | OSM is based on user contributions, so accuracy cannot be guaranteed. Based on limited spot checks, bike lanes seem relatively accurate, but some paths that do not allow bikes were incorrectly marked as allowing them. More effort is needed to handle infrastructure that was entered in an unusual way. |

### Block Group Data
All Block Group data is currently displayed based on 2021 Census Block Groups

| What? | Source | Description | Accuracy Notes |
| :---- | :---- | :---- | :---- |
| “Sustainable Mobility” Modeshare | ACS 5 Year 2021 Table B08301 | The proportion of workers who reported traveling to work by transit, bike, or walking, excluding those who work from home | Varies highly from block group to block group |
| “Carlessness” Rate | ACS 5 Year 2021 Table B25044 | The proportion of occupied housing units that have no vehicles associated | Varies highly from block group to block group |
| Total Population | ACS 5 Year 2021 Table B01003 | The population of the block group |  |
| Population Density | SmartLocationDB | The population density of the block group |  |
| Total Jobs | EPA SmartLocation DB | Number of jobs in the block group, based on 2017 LODES | Data for all SmartLocation values is provided with 2018 block groups, had to impute values based on an area-weighted mean where these do not match 2021 block groups   |
| Job Accessibility by Transit in 45 mins | EPA SmartLocation DB | The number of jobs accessible from the block group by transit with less than a 45 minute trip, based on 2017 LODES data and 2021 transit service |  |
| Job Density | EPA SmartLocation DB | Number of jobs per acre, based on 2017 LODES |  |
| Retail & Entertainment Job Density | EPA SmartLocation DB | Number of jobs in retail and entertainment per 2017 LODES data, specifically those classified in NAICS sectors 44-45 (retail trade), 71 (arts/entertainment), and 72 (food service) | Used as proxy for density of retail and entertainment destinations |
| Jobs / Housing Ratio | Derived from SmartLocation DB | Total number of jobs divided by the number of housing units |  |
| National Walkability Index | EPA SmartLocation DB | Metric for estimating walkability of an area. Based on intersection density, proximity to any transit stop, and diversity of land use in a block group. A score above 10.5 implies “above average” walkability |  |
| National Traffic Proximity & Volume Percentile | US EPA [EJscreen](https://www.epa.gov/ejscreen) | The block group’s percentile of the count of vehicles (AADT, avg. annual daily traffic) at major roads within 500 meters divided by distance in meters (not km) |  |
| Bike Infrastructure Friendliness | Planned based on OSM | Still in refinement, but planning on basing this on the proportion of streets that are “residential” or have bike facilities OSM, and and the intersection density between such streets |  |
